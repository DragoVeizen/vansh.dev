This site is mostly an excuse.

For two years at [Hudle](https://hudle.in) I've been working on backend systems — payments, search, an audit pipeline — and I keep hitting the same loop: build the thing, ship the thing, never write down what I learned. The decisions, the tradeoffs, the half-hour at 11pm where I finally figured out why a feature flag wasn't doing what I expected — it all evaporates.

So I built this site, partly to fix that.

You can expect posts on:

- **Payments.** Re-architecting Hudle's payment pipeline from a single-record model into a 9-table entity split, rolled out behind a feature flag with zero downtime. That long-form is up next.
- **Search and data.** Elasticsearch mapping, ClickHouse for audit, the weird places performance hides.
- **Tooling.** Docker + LocalStack dev environments, observability stacks I've built or replaced.

The aim is to write the kind of post I would have wanted to find six months ago — when I was the one trying to figure these problems out for the first time, and `nothing on the internet quite matched the shape of the problem`.

Thanks for reading. More soon.
